# Micromonolith

Conventions-only skill that installs the micromonolith architecture as a glob-scoped rule (active on `**/*.py`). Describes the architectural pattern independently of any framework — domain isolation, composition root, service interfaces, anti-corruption layer, and authorization model.

For framework-specific implementation, see the Pyramid skill family which depends on this one: **pyramid-developer** (shared conventions), **pyramid-app** (composition root, tests, pshell), and **pyramid-plugin** (module anatomy, `includeme`, service layer, REST endpoints).

Read [templates/micromonolith-rule.md](templates/micromonolith-rule.md) and write it to `.cursor/rules/micromonolith.mdc` in the target project. No placeholder substitution needed — the conventions are the same across all projects following this architecture.
