# Pyramid Core (Framework Features)

Canonical reference for Pyramid's framework extension points — the building blocks that all three developer personas (App, Plugin, Library) use when extending Pyramid's request/response pipeline, configuration system, and event model.

If you're looking for persona-specific guidance:

- **App Developer** — composition root wiring, test infrastructure, pshell snippets → see **pyramid-app**
- **Plugin Developer** — domain modules, service layer, REST endpoint scaffolding → see **pyramid-plugin**
- **Library Developer** — standalone extensions, `includeme` for consumers, settings → see **pyramid-library**

For shared Pyramid conventions (registry, models, authorization, transactions), see the **pyramid-developer** always-on rule.
